Configuring the registry

The wasm.toml manifest lives at the root of every package. It declares the package identity, its dependencies, and the registries it pulls from.

Manifest fields

  • name — reverse-DNS package identifier
  • version — semantic version, must be unique per registry
  • authors — one or more contact strings

Resolution order

  1. Local cache at ~/.wasm/store
  2. Registries declared in the manifest, in order
  3. The default registry, unless --offline is set
Registries are append-only. A version, once published, cannot be overwritten — only yanked. 
[package]
name    = "example.com/hello-world"
version = "0.1.0"
authors = ["Lorem Ipsum <lorem@example.com>"]

[dependencies]
"wasi:http"   = "0.2"
"wasi:cli"    = "0.2"

See the manifest reference for the complete schema, including optional fields like [targets.*] and [features].

Regions

Pages are composed of stacked regions. The primary region uses the canvas surface; secondary regions (supporting data, references, appendices) switch to the white surface. The surface swap signals “this is additional content” — no rules or borders are drawn between regions.

Rules

  • 01The primary region (top of page) sits on canvas. Use it for the main subject.
  • 02Secondary regions sit on surface. The boundary is the surface swap itself — no rule, no border.
  • 03Use full-bleed background swap on wide screens so the boundary reads as a true section break, not a card.
  • 04Maximum two surface swaps per page. Beyond that, switch to a new page or tabs.
  • 05Within a region, use lineSoft for internal subdivisions (table rows, list separators).
22

Motion

Motion is functional: it explains state changes, never decorates them. Most transitions sit between 120–260ms on the standard curve. Anything longer needs a reason.

Easing curves

Standard
cubic-bezier(.2,0,0,1)
Default for state changes — hover, focus, expand.
Entrance
cubic-bezier(0,0,0,1)
Elements arriving on screen — toasts, popovers, modals.
Exit
cubic-bezier(.4,0,1,1)
Elements leaving — dismissed alerts, closed sheets.
Spring
cubic-bezier(.34,1.56,.64,1)
Reserved for direct manipulation feedback — toggles, drag-snap.

Durations

fast120msColor, opacity, focus rings.
base180msDefault transition. Hover, expand-collapse.
slow260msPosition changes, panel slides.
page360msRoute transitions, modal open.

Preview (hover row)

fast · std
base · std
slow · std
page · spring

Rules

  • 01Animate transform and opacity only. Avoid width, height, top.
  • 02Default to base + standard. Pick another token only when intent demands it.
  • 03Pair entrance and exit asymmetrically: enter on entrance, leave on exit, exits 60–80% of the entrance duration.
  • 04Spring is for human-initiated, direct manipulation only. Never for system events.
  • 05Honor prefers-reduced-motion — collapse to instant or to a 60ms cross-fade.

Nested Sidebar

Hierarchical navigation for reference docs. Top-level entries collapse with native <details>; sigils classify each row by kind (command, group, flag, env, etc.).

Anatomy
  • One <details> per group; rotates the chevron via details[open].
  • Children indent 14px with a 1px --c-line-soft guide on the left.
  • Active state uses --c-surface-muted + medium weight — no border, no accent.
  • Trailing meta (counts, tags) sits in a ml-auto slot in 10.5px ink-400 mono.
  • Sigil colour signals kind, never status. Use the categorical palette and pair consistently across the surface.

Anatomy
  • Two depth levels only: top-level entries and one indent (.indent, +12px).
  • Hover lifts ink to --c-ink-900 and tints the left rail to --c-line.
  • Active state uses a full --c-ink-900 rail; the rail is the only marker — never combine with a background.
  • Drive the active state from a scroll-spy (IntersectionObserver with a top rootMargin) so the marker tracks the section currently in view.
  • The rail width is reserved (1.5px transparent border) on every row, so toggling active doesn't shift the label.
