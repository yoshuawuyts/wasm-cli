Overview

The Registry API is a thin JSON layer over the OCI-backed package store. It is the same surface used by the wasm CLI for resolution, search, and publishing. Endpoints follow the /v1 prefix and are versioned by URL.

Guide

Authentication

Read endpoints are unauthenticated. Mutating endpoints (POST, DELETE) require a bearer token issued by your registry administrator. Pass it in the Authorization header.

Example
curl -H "Authorization: Bearer $REGISTRY_TOKEN" \
     https://registry.example.com/v1/packages/detail/example.com/hello-world
Guide

Errors

Errors are returned as a JSON object with a stable code string and human-readable message. HTTP status follows the standard semantics below.

Status Code Meaning
200 ok Request succeeded.
400 invalid_request Malformed parameter or request body.
401 unauthenticated Missing or invalid bearer token.
404 not_found No such package, version, or interface.
409 conflict Version already exists; cannot overwrite.
429 rate_limited Too many requests. Retry after the indicated interval.
500 internal_error Unexpected server-side failure.
Error envelope
{
  "code": "not_found",
  "message": "package 'example.com/missing' was not indexed",
  "request_id": "req_01JABCDE6N3QK7Y2S0V"
}
Guide

Pagination

List endpoints use forward-only cursor pagination. Pass ?cursor=<opaque> to continue from the previous page. The next cursor — if any — is returned in the Link header with rel="next". Default page size is 50; maximum is 200.

Health

Service health

Liveness probe for orchestrators and uptime monitors. Cheap; no database access.

GET /v1/health

Returns 200 OK with a fixed payload when the process is reachable.

Responses
  • 200 Service is up.
Request
curl https://registry.example.com/v1/health
Response · 200
{ "status": "ok" }
Packages

Packages

Inspect packages, list versions, and read per-version metadata. Repository paths are case-sensitive and use forward slashes; the registry component is the upstream domain.

GET /v1/packages/detail/{registry}/{*repository}

Resolve a single repository to its full PackageDetail, including every known version, manifest digest, and extracted WIT dependency graph.

Path parameters
registry
string · required
 Upstream registry host, e.g. example.com.
repository
path · required
 Slash-separated repository path. May contain multiple segments — captured as a single greedy parameter.
Responses
  • 200 PackageDetail
  • 404 Repository is not indexed.
Request
curl https://registry.example.com\
/v1/packages/detail/example.com/hello-world
Response · 200
{
  "registry":   "example.com",
  "repository": "hello-world",
  "latest":     "0.1.0",
  "versions": [
    {
      "version": "0.1.0",
      "digest":  "sha256:8c7f…",
      "size":    128944,
      "published_at": "2026-04-01T10:22:18Z"
    }
  ]
}
GET /v1/packages/versions/{registry}/{*repository}

List every published version for a repository, ordered newest first. Each entry is a PackageVersion with manifest metadata.

Query parameters
limit
int · optional
 Maximum results. Default 50, max 200.
cursor
string · optional
 Opaque continuation token from a previous page.
include_yanked
bool · optional
 If true, include yanked versions. Default false.
Responses
  • 200 Array of PackageVersion.
  • 404 Repository is not indexed.
Request
curl "https://registry.example.com\
/v1/packages/versions/wasi/http?limit=5"
Response · 200
[
  {
    "version": "0.2.4",
    "digest":  "sha256:1f08…",
    "yanked":  false
  },
  {
    "version": "0.2.3",
    "digest":  "sha256:90ab…",
    "yanked":  false
  }
]
GET /v1/packages/version/{registry}/{version}/{*repository}

Resolve a single tag to its PackageVersion. The version segment accepts a SemVer tag or the literal latest.

Responses
  • 200 PackageVersion
  • 404 No such tag for this repository.
POST /v1/packages/{registry}/{*repository} Auth required

Publish a new version. The body references an OCI manifest already uploaded via the standard distribution endpoints.

Request body
version
string · required
 SemVer tag, e.g. 0.2.4.
digest
string · required
 Manifest digest, prefixed with sha256:.
signature
string · optional
 Detached cosign-compatible signature, base64 encoded.
Responses
  • 201 Version accepted; returns the canonical PackageVersion.
  • 409 A version with this tag already exists.
Request
curl -X POST \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "version": "0.2.4",
    "digest":  "sha256:1f08…"
  }' \
  https://registry.example.com\
/v1/packages/wasi/http
Response · 201
{
  "version": "0.2.4",
  "digest":  "sha256:1f08…",
  "yanked":  false,
  "published_at": "2026-04-18T08:11:02Z"
}
DELETE /v1/packages/version/{registry}/{version}/{*repository} Auth required

Mark a version as yanked. Yanked versions remain resolvable by exact digest but are excluded from version listings unless include_yanked=true is set. This operation is reversible by re-publishing the same digest.

Responses
  • 204 Version yanked.
  • 401 Token missing or invalid.
  • 404 No such tag.
Search

Interface search

Find packages by the WIT interfaces their worlds import or export. Interface identifiers use the canonical namespace:package/interface@version form.

GET /v1/search/by-import

Return packages whose worlds import the given interface.

Query parameters
interface
string · required
 Interface identifier, e.g. wasi:http/incoming-handler@0.2.0.
Responses
  • 200 Array of Package.
  • 400 Malformed interface identifier.
Request
curl "https://registry.example.com\
/v1/search/by-import\
?interface=wasi:http/incoming-handler@0.2.0"
Response · 200
[
  {
    "registry":   "example.com",
    "repository": "hello-world",
    "latest":     "0.1.0"
  },
  {
    "registry":   "fermyon.com",
    "repository": "spin-http",
    "latest":     "3.1.2"
  }
]
GET /v1/search/by-export

Return packages whose worlds export the given interface. Mirrors by-import in shape and parameters.

Responses
  • 200 Array of Package.
  • 400 Malformed interface identifier.
