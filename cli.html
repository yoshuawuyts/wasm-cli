v2.4.0 CLI Reference Command group

wasm registry

Manage the registries the toolchain publishes to and resolves dependencies from — add, remove, authenticate, and publish.

Since 1.2.0
Source cli/registry.rs
Reference man wasm-registry(1)
Status Stable

Manage the registries wasm publishes to and resolves dependencies from. A registry maps a namespace (e.g. wasi:) to a backing OCI-compatible store. wasm registry is a command group: it has no behaviour of its own — invoking it without a subcommand prints help and exits with status 2.

Registry configuration is layered: project-local files override user-level files, which override system defaults. Use wasm registry list to see the effective set, and wasm registry add to register a new one.

SYNOPSIS 
wasm registry <SUBCOMMAND> [FLAGS] [ARGS...]

# Common forms
wasm registry add     <name> <url>
wasm registry remove  <name>
wasm registry list    [--scope <scope>]
wasm registry login   <name> [--token <token>]
wasm registry publish [<path>] [--registry <name>]

wasm registry add§

1.2.0 source man

Register a new namespace and point it at an OCI registry URL.

wasm registry add <name> <url> [--scope <scope>] [--default] [--insecure]

The namespace must be a bare identifier (e.g. wasi, acme) and becomes the prefix used in package coordinates like acme:my-component@1.0.0.

Arguments
namestring · required
Namespace identifier. Must match [a-z][a-z0-9-]{0,38}. Reserved: wasi, wasm, component.
urlurl · required
OCI-compatible registry endpoint, e.g. https://ghcr.io/acme. The https:// scheme is required unless --insecure is passed.
Options
--scope <scope>user · project · system
Where to write the registry entry. Defaults to user. project requires a wasm.toml.
--defaultflag
Make this registry the default target for publish.
--insecureflag · danger
Allow plaintext http:// URLs and skip TLS verification. Use only against trusted local mirrors.
--token <token>string
Authenticate immediately after adding. Equivalent to running wasm registry login afterwards. Prefer reading from $WASM_REGISTRY_TOKEN for unattended setup.
Examples
# Add a public registry under the "acme" namespace
$ wasm registry add acme https://ghcr.io/acme
added registry acme → https://ghcr.io/acme (scope: user)
# Add a project-local mirror and make it the default
$ wasm registry add internal https://oci.corp/wasm --scope project --default
added registry internal → https://oci.corp/wasm (scope: project)
default registry set to internal
Caveat Adding a registry does not verify its reachability. Run wasm registry list --check to probe the configured endpoints.

wasm registry remove§

1.2.0 source man

Forget a registered namespace. Locally cached artifacts are kept by default.

wasm registry remove <name> [--scope <scope>] [--purge]

Removes the entry for name from the configuration scope. Cached artifacts remain on disk so that pinned, already-resolved dependencies continue to build offline.

Options
--scope <scope>user · project · system
Scope to remove from. Defaults to the highest-priority scope that defines name.
--purgeflag
Also delete cached artifacts under $WASM_CACHE_DIR/registry/<name>/.
Example
$ wasm registry remove internal --purge
removed registry internal
purged 142 cached artifacts (348 MB freed)

wasm registry list§

1.2.0 source man

Print the effective set of registries, with the source of each entry.

wasm registry list [--scope <scope>] [--check] [--format <fmt>]

Entries are sorted by precedence (project > user > system). When the same namespace is defined in multiple scopes, only the winning entry is printed unless --all is given.

Options
--scope <scope>user · project · system
Restrict output to one scope.
--allflag
Show shadowed entries too, with the scope marked.
--checkflag
Probe each endpoint and report reachability and auth state.
--format <fmt>table · json · toml
Output format. json and toml are stable for scripting.
Output
$ wasm registry list --check
NAME       URL                              SCOPE     STATUS
internal   https://oci.corp/wasm            project   ok     authenticated
acme       https://ghcr.io/acme             user      ok     anonymous
wasi       https://wa.dev/wasi              system    ok     anonymous
mirror     http://10.0.0.4:5000             user      fail   connection refused

wasm registry login§

1.4.0 source man

Store credentials for a registry in the OS keychain.

wasm registry login <name> [--token <token>] [--username <user>] [--password-stdin]

Credentials are stored in the platform keychain (Keychain on macOS, Credential Manager on Windows, libsecret on Linux). If no keychain is available, wasm falls back to a mode-600 file under $WASM_CONFIG_DIR/credentials.toml and warns once.

Options
--token <token>string
Bearer token. Mutually exclusive with --username.
--username <user>string
Username for basic auth. Password is prompted on a TTY.
--password-stdinflag
Read the password or token from stdin instead of prompting.
Examples
# Interactive token prompt
$ wasm registry login acme
Token (input hidden): ****
ok stored credentials for acme in macOS Keychain
# Unattended (CI)
$ echo "$REGISTRY_TOKEN" | wasm registry login acme --password-stdin
ok stored credentials for acme

wasm registry logout§

1.4.0 source man

Remove credentials for a registry from the OS keychain.

wasm registry logout <name> [--all]

Pass --all to clear credentials for every configured registry. The registry entry itself is preserved — only the secret is dropped.

Example
$ wasm registry logout acme
ok cleared credentials for acme

wasm registry publish§

1.3.0 source man

Build and upload the current package to a registry.

wasm registry publish [<path>] [--registry <name>] [--tag <tag>] [--dry-run] [--allow-dirty]

Builds the package at path, produces an OCI artifact, and pushes it to the target registry. The version published is the one declared in wasm.toml; re-publishing the same version is rejected unless --overwrite is set on a registry that allows it.

Publishing is performed in three stages — build, sign, push — each of which can be inspected separately with --stage.

Arguments
pathpath · optional
Directory containing a wasm.toml. Defaults to ..
Options
--registry <name>string
Target registry. Defaults to the one marked default = true.
--tag <tag>string · repeatable
Additional OCI tag to push (e.g. latest, canary).
--stage <stage>build · sign · push
Run only the named stage. Useful for splitting CI jobs.
--dry-runflag
Print what would be uploaded without contacting the registry.
--allow-dirtyflag
Publish even when the working tree has uncommitted changes.
--overwriteflag · danger
Replace an existing version on registries that permit it.
Example
$ wasm registry publish --registry acme --tag latest
[1/3] build      ✓  acme:hello@0.4.2 (1.2 MB)
[2/3] sign       ✓  cosign · 1 signature
[3/3] push       ✓  ghcr.io/acme/hello → sha256:8c4a…f12
published acme:hello@0.4.2 + tag latest in 4.1s
Tip Pair --dry-run with --format json to feed the planned upload manifest into review tooling before publishing.

Exit codes §

0
Success.
1
Generic failure. The error message is printed to stderr.
2
Usage error — invalid flags, missing arguments, or a command group invoked without a subcommand.
3
Configuration error — malformed wasm.toml or unreachable registry.
4

130

Config files §

Registry settings persist as TOML. The schema is identical across scopes; only the search location differs.

$WASM_CONFIG_DIR/registries.toml 
# One [registry.<name>] table per registered namespace.
# The "default" key may appear on at most one entry.

[registry.acme]
url     = "https://ghcr.io/acme"
default = true

[registry.wasi]
url = "https://wa.dev/wasi"

[registry.internal]
url      = "https://oci.corp/wasm"
insecure = false
scope    = "project"     # informational; set by --scope
See also manifest reference, credentials, resolution.
